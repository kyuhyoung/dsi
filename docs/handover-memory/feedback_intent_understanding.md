**원칙 (사용자 확정 2026-05-25, 근본 치유 7차)**: *"양식의 어느 것도 아무 이유 없이 존재하지 않는다. 모든 요소는 의도가 있다."*

**Why**: 시스템이 fills 만들 때 각 셀을 *독립적으로* 채움 → 모순 산출. 예: "단독신청 (다비오)" 적었는데 "컨소시엄 구성 여부 □여 □부" 는 둘 다 빈 칸 + "컨소시엄 참여기업 (0社) A주식회사" placeholder 그대로. 사람이 보면 즉시 *논리 모순* 인식.

**How to apply — 양식 요소 의도 분류**:

| 요소 | 의도 | 시스템 행동 |
|---|---|---|
| `※ ... 작성` 박스 | 작성 방법 지침 | **건드리지 마라** (양식 그대로) |
| `[별지 제N호] ...` | 별지 헤더·구조 | 분리 단위 인식 |
| `1-1. 상용화 대상 소개` 같은 섹션 헤더 | 본문 작성 자리 표시 | 그 *다음* 본문 placeholder에 실제 내용 |
| `○ 가나다 / - 가나다 / * 가나다` | 3-tier bullet 작성 안내 | bullet 구조 유지 + 실제 내용으로 교체 |
| `□ 여 / □ 부` 같은 체크박스 | 분기 결정 | 비즈니스 결정 후 *적절히 ☑* |
| `↳ ...` 종속 셀 | 상위 답에 따른 조건부 | 상위 결정 후 *채움 or 비움/생략* |
| `(0社) A주식회사, B주식회사 ...` 예시 | 형식 안내 | *실제값으로 교체* 또는 "해당 없음" |
| `0000.00.00` 같은 날짜 placeholder | 형식 안내 | 실제 날짜로 교체 |
| `OO법인 주식회사` 같은 이름 placeholder | 형식 안내 | 실제 회사명으로 교체 |
| 라벨 셀 (음영) + 입력 셀 (옆) | 메타 필드 | 짧은 값 |
| 큰 빈 셀 (라벨 없음 + 큰 공간) | 본문 작성 자리 | 긴 본문 |

**일관성 룰 (상호 의존 셀 그룹)**:
- 비즈니스 결정 셀들 (단독/컨소, 타입1/2, 신청금액 등) 을 *먼저 고정*
- 그 결정에 따라 *종속 셀 채움* (참여기업·자기부담금 등)
- 모순 0 보장 — 본문에 "단독신청" 적으면 □부 ☑ + 참여기업 = "-"

**implementation 단계**:
1. extract: 양식 요소 *의도 자동 분류* (작성지침/placeholder/체크박스/메타/본문/조건부)
2. proposal-writer: *비즈니스 결정 먼저* → 의도 기반 일관 채움
3. fill: 조건부 셀 그룹 처리 + 체크박스 ☑ 교체

**일반화**:
- 의도 분류 룰은 *한국 공공 RFP 양식 표준 패턴* — 도메인 무관
- 농식품AI 특화 X. 임의 양식에 동작.
