**원칙 (사용자 확정 2026-05-25, 근본 치유 6차)**: *"양식의 placeholder 단락 ('가나다' 등) 을 실제 본문으로 자동 생성·교체. 이게 AI 자동화의 핵심 가치."*

**Why**: ideal D (별지 제3호 사업계획서) 의 본체는 *표가 적고 본문이 많음*. 양식엔 "○ 가나다 / - 가나다 / * 가나다" 같은 작성 자리 표시가 박혀있고, 사람은 그 자리에 *5~10페이지 실제 사업 추진방안 본문*을 적음. 셀 빈 칸 채움만으론 본체 별지 80% 가 *"가나다" 그대로* 남음. AI 자동화의 결정적 가치 = *이 본문 자동 생성*.

**How to apply**:
- **양식의 채움 대상 = 2종**: (a) 빈 셀 (메타·짧은 값) + (b) **placeholder 단락 (긴 본문)**.
- **placeholder 단락 검출 패턴 (한국 RFP 일반)**:
  - 더미 텍스트: "가나다", "ㅇㅇ", "XX", "(작성)", "(예시)"
  - 또는 *섹션 헤더 직후 빈 또는 더미 단락*
  - 정규식 후보: `^\s*[○\-\*·]\s*가나다\s*$`, `^\s*(가나다|ㅇㅇ|XX)\s*$`, `(작성|내용|여기에).*작성`
  - 안내 박스 (※ 표기) 는 *건드리지 마라* — 작성자 지침
- **본문 단락 생성 (proposal-writer)**:
  - 각 placeholder 단락의 *섹션 헤더* (직전 hp:p 텍스트, 예: "1-1 상용화 대상 소개") 가 *작성 주제*
  - 직전 안내 박스 (※ 작성 방법) 가 *작성 가이드*
  - 두 가지를 LLM 컨텍스트로 → KB + RFP 활용해 *수십 줄 본문* 생성
- **단락 교체 (fill_hwpx_form 확장)**:
  - placeholder hp:p 의 *hp:run > hp:t 텍스트 교체* 또는 hp:p 통째 새 트리로 교체 (multi-line text → 다중 hp:p)
  - 한컴 PDF render 빈 셀 버그와 다를 수 있음 — 본문 단락은 *원래 비빈 hp:t* 라 hp:t 교체 OK 가능성 (검증 필요)
- **일반화**: placeholder 패턴은 *양식별 가변*. 양식 분석 시 *자동 검출* 또는 *사용자 review*. 농식품AI "가나다" 하드코딩 X.
