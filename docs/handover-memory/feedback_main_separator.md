**원칙 (사용자 확정 2026-05-25, 근본 치유 5차)**: *"산출물은 본체 별지 1개. 부속 별지에 채움 시간 낭비 X."*

**Why**: 근본 치유 3·4차에서 통합 양식 .hwp/.hwpx 의 661 빈 셀 모두 채우려 함 → 자가진단 ○/× 박는 한컴 PDF 렌더 버그까지 깊이 파고들었음. 본질 놓침. ideal D 파일명 `[2차 취합] [별지 제3호] 사업계획서_최종본2(인건비 제외)(공유용).hwp` 가 결정적 단서 — *별지 3 사업계획서만 별도 파일*. RFP는 별지별 분리 작성·제출 요구.

**How to apply**:
- **본체 별지 식별 = RFP 명시 따름**: `rfp-analyst` yaml 의 `사업개요.본체_별지` 사용. 농식품AI 케이스에선 "별지 제3호 사업계획서". *하드코딩 금지* — RFP가 "제안서" "기획서" 같이 다르게 명명하면 그것 따름.
- **양식 분할 = 별지 마커 패턴 매칭**: 양식 셀/단락 텍스트에서 `\[별지\s*제?\s*\d+\s*호?\]` 또는 `[붙임\s*\d+]` 같은 한국 공공 RFP 표준 패턴 매칭. 매치 위치 = 별지 경계. 농식품AI 특화 X.
- **fill 범위 = 본체 별지 셀만**: form.yaml 에 `sections: [{label, table_idxs}]` 명세 추가. proposal-writer 가 본체 별지 셀만 fills 작성. 부속 별지는 빈 채.
- **산출물 = 본체 별지만 담긴 .hwpx**: split_hwpx_section.py 또는 fill 시 부속 별지 트림. ideal D 같은 단일 파일.
- **부속 별지 처리는 별도**: 자가진단 ○/× 같은 단순 채움은 *옵션 산출물*. 사용자가 한컴 GUI 로 손쉽게 채움. 시스템이 *자동화 안 해도 OK*.
