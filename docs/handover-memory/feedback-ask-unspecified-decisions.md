사용자가 명시하지 않은 *전략적·비즈니스 결정*(신청유형, 신청 금액, 컨소시엄 구성, 사업 범위 등)은 **임의로 가정하거나 기본값을 박지 말고, 반드시 사용자에게 물어 확정한다.**

**Why**: 이런 결정은 사업 결과·심사 통과에 직결되는 *사용자(신청 기업)의 권한·책임*이다. 잘못 가정하면 제안 전체(예산·일정·증빙)가 어긋난다. *이전 세션/버전의 결정을 근거 없이 답습*하는 것도 가정에 해당 — 재확인해야 한다. "합리적으로 보이니 일단 X로" 진행 금지.

**How to apply**: 작성 중 이런 항목을 만나면 `(확인 필요)` 표기 + 사용자에게 질문(AskUserQuestion). 근거를 댈 수 있으면 제시하되 *확정은 사용자*. 예: 농식품AI 제안에서 신청유형(타입1=1년 내 상용화·국고20억 / 타입2='27까지·국고10억)을 묻지 않고 타입1로 답습한 게 잘못 — 타입은 상용화 성숙도 증빙·기간 전략에 따른 사용자 결정. 같은 양식 채움 셀 단위 의도는 [[feedback_intent_understanding]], overfit 금지는 [[feedback_no_overfit]].
