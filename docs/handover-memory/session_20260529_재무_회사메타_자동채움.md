## 시스템 구조 (이번 라운드 완성)

```
kb/company/<name>/
├── intro.md                   ← 마크다운 회사 소개
├── profile.yaml               ← 회사 메타 구조화 (이번 라운드 추가)
├── finance.yaml               ← 재무제표 yaml (이번 라운드 추가)
└── finance/
    └── *.xlsx                 ← 재무제표 원본 (사용자 제공)

templates/
├── profile_schema.yaml        ← 회사 메타 표준 스키마
├── profile_label_map.yaml     ← 양식 라벨 ↔ 회사 메타 필드 매핑
├── finance_schema.yaml        ← 재무 표준 스키마
├── finance_label_map.yaml     ← 양식 라벨 ↔ 재무 필드 매핑
└── system_defaults.yaml       ← 빌더 정책 (단위·threshold 등)

scripts/
├── extract_finance.py         ← xlsx → finance.yaml (자동)
├── fill_finance_cells.py      ← form.yaml + finance.yaml → fills (재무)
├── fill_company_cells.py      ← form.yaml + profile.yaml → fills (메타)
├── extract_hwpx_form.py       ← 양식 분석 (정규식 yaml 정본화 완료)
├── fill_hwpx_form.py          ← fills.yaml → 채워진 hwpx
└── split_hwpx_by_section.py   ← 통합 → 별지별 (분할 가드 적용)
```

## 데이터 흐름 (재현 단계)

```bash
# 1. 회사 재무제표 xlsx → finance.yaml
python scripts/extract_finance.py \
    "kb/company/dabeeo/finance/Dabeeo_표준재무제표_FY23-25_FIN.xlsx" \
    "kb/company/dabeeo/finance.yaml" \
    --company "주식회사 다비오"

# 2. 양식 분석 (한 번만, 결과 재사용 가능)
python scripts/extract_hwpx_form.py \
    "output/20260526/농식품AI_양식.hwpx" \
    "output/20260529/통합양식.form.yaml"

# 3. 회사 메타 fills 생성
python scripts/fill_company_cells.py \
    "output/20260529/통합양식.form.yaml" \
    "kb/company/dabeeo/profile.yaml" \
    "output/20260529/fills_profile.yaml"

# 4. 재무 fills 생성
python scripts/fill_finance_cells.py \
    "output/20260529/통합양식.form.yaml" \
    "kb/company/dabeeo/finance.yaml" \
    "output/20260529/fills_finance.yaml"

# 5. fills 병합 (회사 메타 우선)
python -c "
import yaml
f1 = yaml.safe_load(open('output/20260529/fills_profile.yaml', encoding='utf-8')).get('fills', [])
f2 = yaml.safe_load(open('output/20260529/fills_finance.yaml', encoding='utf-8')).get('fills', [])
seen, merged = set(), []
for f in f1 + f2:
    if f['id'] not in seen:
        seen.add(f['id'])
        merged.append(f)
with open('output/20260529/fills_merged.yaml', 'w', encoding='utf-8') as fh:
    yaml.dump({'fills': merged}, fh, allow_unicode=True, sort_keys=False)
print(f'merged: {len(merged)}')
"

# 6. hwpx 채움
python scripts/fill_hwpx_form.py \
    "output/20260526/농식품AI_양식.hwpx" \
    "output/20260529/fills_merged.yaml" \
    "output/20260529/농식품AI_v15.hwpx"

# 7. 별지 분할
python scripts/split_hwpx_by_section.py \
    "output/20260529/농식품AI_v15.hwpx" \
    "output/20260529/통합양식.form.yaml" \
    "output/20260529/별지_v15"

# 8. PDF + PNG (시각 검증)
python scripts/hwpx_to_pdf.py \
    "output/20260529/별지_v15/03_[별지_제2-2호]_사업신청서.hwpx" \
    "output/20260529/별지_v15/03_2-2.pdf"
python scripts/pdf_to_png.py \
    "output/20260529/별지_v15/03_2-2.pdf" \
    "output/20260529/v15_2-2_pages" 110
```

## 이번 라운드 진단·치유 흐름 (핵심)

1. **재무 자동 채움 (별지 제2-2호 2.재무현황 표)** — finance_schema·label_map·extract_finance·fill_finance_cells 구축. 18셀 매칭.
2. **회사 메타 자동 채움 (1.기업정보 표)** — sample directive "대표 이름·사업자번호 시키지 않아도 찾아야". profile_schema·label_map·fill_company_cells. 44셀 매칭.
3. **유상증자 (투자유치)** — 사용자 지적 "재무제표에 투자 실적은 안 적혀 있디?". 현금흐름표 r27 자동 추출.
4. **세로형 표 처리** — T7 매출액 (3행 × 3년) 처리. row label vertical merge 보정 + own_row_year 추출.
5. **모든 셀에 hints 생성 (extract_hwpx_form)** — example/label_or_content 셀의 *값 자리*도 KB lookup 가능하게.
6. **`_own_texts(p)` 룰 (cleanup nested 표 텍스트 흡수 버그 수정)** — outer paragraph 의 `iter("hp:t")` 가 descendant 까지 흡수 → 표 안 cell 텍스트 모두 비우는 버그. 직접 자식 hp:run 의 hp:t 만 검사로 수정.
7. **별지 라벨 박스 *유지* 정책 (사용자 정정)** — v8 에서 *박스 제거* 가드가 *데이터 손상*. v14 에서 폐기 + 모든 표 height-release 로 빈공간만 압축.
8. **단위 백만원 통일 정책 (사용자 명시)** — `proposal_output_unit: 백만원` yaml 정본. 양식 단위 명시 없는 셀은 백만원 통일. 양식 옆 단위 셀 있으면 그것 우선.
