# 한국 공공 RFP 분석 패턴

## 표준 항목 체크리스트

### 사업 개요
- 사업명
- 발주기관 (조달청, 한국공항공사, 한국전력공사 등)
- 사업 기간 (계약일 ~ 준공일)
- 사업 예산 (VAT 포함/미포함 명시 여부 확인)
- 사업 분류 (정보화 / SI / 유지보수 / 연구개발 등)

### 사업 내용·범위
- 추진 배경·목적
- 사업 범위 (시스템·기능·지역·기간)
- 주요 산출물·납품 형태

### 자격 요건
- 입찰참가자격 (기업 등급, 인증, 실적 요건)
- 컨소시엄 구성 가능 여부
- 신용평가 등급
- 보유 기술자 자격 요건

### 기술 요구사항
- 기능 요구사항 (FUR)
- 비기능 요구사항 (보안·성능·가용성)
- 표준 준수 (전자정부 표준 프레임워크, KS 인증 등)
- 호환성·연동 요구사항

### 평가 항목·배점
- 기술평가 (보통 80~90점)
- 가격평가 (보통 10~20점)
- 세부 평가항목별 배점·정성/정량 구분

### 제출 요건
- 제출 기한 (입찰 마감일·시간)
- 제출 방법 (나라장터·우편·방문)
- 제출 서류 목록
- 양식 지정 여부 (별첨 양식 사용 의무 여부)

## 누락 항목 검출 (확인 요청 대상)

다음 항목이 RFP에 빠져 있으면 발주처에 사전 질의 권장:
- 평가배점 세부 항목
- 사업비 산출 기준
- 변경관리 절차
- 검수 기준
- 하자보수 조건·기간

## 위험 신호 (사업 리스크 표지)

- *"기술협상 후 결정"* — 계약 후 범위 변경 우려
- *"발주처 요구에 따라 변경 가능"* — 범위 무한 확장 가능성
- 비현실적으로 짧은 일정 — 사전 작업 가능성 (이미 내정된 RFP 형식화 의심)
- 매우 구체적인 자격 요건 — 특정 업체 맞춤 가능성

## 출력 형식

RFP 분석 결과는 다음 구조로 정리:

```yaml
사업개요:
  사업명: ...
  발주기관: ...
  사업기간: ...
  예산: ...

요구사항:
  기능: [...]
  비기능: [...]
  표준준수: [...]

평가배점:
  기술: 80
  가격: 20
  세부항목:
    - 항목: 추진방안
      배점: 30

제출요건:
  마감일: ...
  방법: ...
  서류: [...]

누락_항목: [...]
위험_신호: [...]
```
