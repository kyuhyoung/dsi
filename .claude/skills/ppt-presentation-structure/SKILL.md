# 발표자료 슬라이드 표준 구성

## 표준 슬라이드 순서

```
 1. 표지 (사업명·발주처·제안사·일자)
 2. 목차
 3. 회사 소개 (1~2장)
 4. 사업 이해
    4-1. 사업 배경·목적
    4-2. 추진 범위
 5. 추진 방안
    5-1. 추진 전략
    5-2. 핵심 방법론
    5-3. 단계별 접근
 6. 추진 일정 (간트차트)
 7. 수행 조직 (조직도 + 핵심 인력)
 8. 사업 수행 역량
    8-1. 유사 실적
    8-2. 보유 기술
 9. 품질·보안 관리
10. 결론·요청사항
11. Q&A
12. 부록 (선택)
```

## 분량 가이드

| 항목 | 공공 사업 | 민간 사업 |
|---|---|---|
| 총 슬라이드 수 | 20~30장 | 15~25장 |
| 발표 시간 | 슬라이드당 1~1.5분 | 슬라이드당 1~1.5분 |
| 슬라이드당 핵심 메시지 | 3개 이내 | 3개 이내 |
| 슬라이드당 본문 글자 | 100자 이내 | 100자 이내 |

## 평가배점 ↔ 슬라이드 분량 매핑

발표 시간과 슬라이드 수는 평가배점 비율에 맞춰 배분:

- 평가배점 큰 항목 = 슬라이드 수 ↑, 발표 시간 ↑
- 예: "추진방안" 30점 → 슬라이드 7~10장, 발표 10~12분
- 예: "회사소개" 5점 → 슬라이드 1~2장, 발표 1~2분

## 시각화 원칙

### 색상
- 3색 이내 (회사 CI 색 + 강조색 1개 + 중성색)
- 발주처 CI 색상은 *오해 방지*를 위해 사용 지양

### 폰트
- 2종 이내 (제목용·본문용)
- 발표장 뒷자리 가독성 기준 최소 18pt 이상

### 시각 요소
- 텍스트 위주 슬라이드 지양
- 다이어그램·플로우차트·표·이미지 적극 활용
- 모든 차트·표는 캡션 + 출처 명시

## 자주 하는 실수

- **빽빽한 텍스트** — 발표자가 읽는 슬라이드와 청중이 읽는 슬라이드가 충돌
- **작은 차트·표** — 뒷자리에서 안 보임
- **제안서 본문을 그대로 슬라이드에 옮김** — 재구성 필요 (본문 ≠ 슬라이드)
- **결론·요청사항 누락** — 마지막에 *"다음 단계로 무엇을 해주셔야 하는지"* 명시 필요
- **Q&A 슬라이드만 표지처럼 단독 배치** — 예상 Q&A 슬라이드를 부록에 준비해두면 가산점

## 출력 형식

ppt-designer agent가 이 skill을 참조해 슬라이드 구성안을 만들 때 사용할 형식:

```yaml
슬라이드:
  - 번호: 1
    제목: "표지"
    내용:
      - 사업명: ...
      - 발주처: ...
      - 제안사: 다비오
      - 제출일: ...

  - 번호: 5-1
    제목: "추진 전략"
    핵심메시지:
      - "..."
      - "..."
    시각요소: "전략 다이어그램 (3단계)"
    발표시간_분: 2.5
```
