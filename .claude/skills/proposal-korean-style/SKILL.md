# 한국어 제안서 작성 스타일

> 본 스킬은 *한국 공공·민간 RFP 공통* 작성 규칙. 사업별 전용 매핑(`meta`·`proposal_structure`·`form_fill_rules` 등)은 `output/learned_patterns_<사업명>.yaml` 참조.

## 기본 톤

- **격식체 "당사는~합니다"** (반말·평어 금지)
- 1인칭은 "저희"보다 "당사" 선호
- 발주처 호칭: "귀 기관", "발주처", 또는 정식 명칭
- 수동태보다 능동태
  - "~에 의해 수행되어진" ❌
  - "당사가 수행함" ⭕

## 종결 어미 — 격식체 + 명사형 종결

공공 사업계획서·제안서는 **개조식·명사형 종결 "~함"·"~임"** 비중이 격식체 "~합니다"보다 훨씬 높음.

| 패턴 | 사용 위치 |
|---|---|
| `~함` (완료·확정형) | 본문 평서문 대부분 |
| `~임` (정의·상태) | 정의·요건 진술 |
| `~합니다` | 표지·인사말·요약 |
| `~할 계획임`·`~예정임` | *향후 계획*에만 사용 |

**예시**:
- "F1-Score 0.98 수준의 고정밀 탐지 성능을 확보**함**."
- "사업 운영과정에서 축적된 데이터 및 기술 노하우를 기반으로 실증 중심의 기술 경쟁력을 확보**함**."
- "IPO를 통해 ~ 재투자 **예정**" (계획)

### 단정형 ≫ 추측형

- "~할 것이다", "~할 수도 있다" 같은 추측형 *사용 금지*
- "~함", "~확보함", "~달성함" 등 **완료·확정형** 우선
- 작성요령에 "추상적 표현 금지"가 명시되면 반드시 단정형

## 1인칭 — "당사" 통일

- 회사 자칭은 **"당사"** 로 통일 (제안서 전체에서 일관)
- 회사 *공식명*(예: "다비오")은 *제품·플랫폼명 앞*에서만 사용 (예: "다비오 Eartheye 플랫폼")
- 사업 자칭은 "본 사업"·"동 지원과제"

## 문장 구조

- 한 문장에 하나의 의도 (긴 복문 지양)
- 결론 먼저, 근거 나중 (PREP: Point–Reason–Example–Point)
- 능동·짧고 명확하게

## 3-tier Bullet 구조 (❍ / - / *)

공공 RFP 양식의 자리표시자 `❍ 가나다 / - 가나다 / * 가나다`를 본문 전반에 일관 적용:

```
❍ (대주제: 배경·필요성·기능 등)
  - (소주제: 구체 항목 또는 근거)
    * (세부 설명: 정량 데이터·사례·메커니즘)
```

**예시**:
```
❍ 시장 선정 배경
  - 전략적 중요성
    * 세계 시장의 약 35%를 차지하며 ...
  - AI 기반 전환 수요 확대
    * 공급 구조 불안정과 ...
```

**규칙**:
- 양식이 별도 구조를 명시하지 않으면 *3-tier bullet 기본 적용*
- 들여쓰기 일관 (2 space)
- `*` 세부 설명은 1~2문장 단순 구조 유지
- *기술 배경* 같은 깊은 설명은 *장문 단락*으로 별도 처리 (3-tier 밖)

## 강조 표현 (Signature Phrases)

### "당사는 ~을 보유/확보/구축함" 패턴

핵심 자산 강조 시 반복 사용:

| 패턴 | 용도 |
|---|---|
| `~을 보유하고 있음` | 자산·기술·인프라 |
| `~을 확보함` | 실적·인증·특허 |
| `~을 구축함` | 시스템·조직·체계 |
| `~을 입증함` | 성능·신뢰성 |

### 우월성 표현 — *근거 동반 필수*

"세계 최초", "국내 최초", "최고 수준" 같은 표현은 *반드시 근거·시점·범위 한정* 동반:

- ❌ "세계 최고 수준의 기술"
- ⭕ "2023년 세계 최초로 30cm 초고해상도 위성영상을 AI로 분석한 사례"

근거 없는 마케팅 우월성 표현은 *지양*. 단 객관 사실로 입증 가능하면 *시점·수치 동반해 사용*.

## 정량 수치 동반 원칙

거의 모든 강조 표현에 **정량 수치 동반** — 단순 수식어 회피.

| 회피 | 권장 |
|---|---|
| "고정밀 탐지" | "F1-Score 0.98 수준의 고정밀 탐지" |
| "다수의 특허 보유" | "등록 25건·출원 26건, 총 51건의 특허 자산 확보" |
| "대규모 투자 유치" | "누적 약 369억원 규모의 투자 유치" |
| "다수의 정부지원 수행" | "정부지원 29건·112.4억원" |

**규칙**:
- 정성 주장(고정밀·대규모·다수) 단독 사용 *금지*
- 수치는 *KB 검증값* 우선. KB에 없으면 부록 *"확인 필요 항목"*

## "End-to-End"·"통합"·"고도화" 키워드

회사 차별점을 표현할 때 자주 활용되는 키워드 패턴:

- "데이터 처리부터 분석, 업무 활용까지 이어지는 엔드투엔드(End-to-End) 서비스 구조"
- "데이터 수집–정제–AI 분석–결과 제공까지 전 과정을 통합한 서비스 구조"
- "기술 고도화"

회사 KB(`dabeeo-profile`) 핵심 가치를 *본문에 그대로 차용* 가능.

## 표준 구조 (공공 제안서)

RFP가 양식을 명시하지 않은 경우의 기본 구조:

```
1. 사업 이해
   1.1 사업 배경
   1.2 사업 목적
   1.3 사업 범위

2. 사업 추진 방안
   2.1 추진 전략
   2.2 추진 방법론
   2.3 단계별 접근

3. 추진 일정·조직
   3.1 추진 일정 (WBS, 간트차트)
   3.2 수행 조직도
   3.3 핵심 인력 이력

4. 사업 수행 역량
   4.1 회사 소개
   4.2 유사 실적
   4.3 보유 기술

5. 품질·보안 관리
   5.1 품질관리 방안
   5.2 정보보안 관리
   5.3 변경관리 절차

6. 부록
   - 인증서 사본
   - 기술 자료
   - 회사 일반 자료
```

RFP가 양식·섹션 순서를 명시하면 *RFP 명시 구조가 최우선*.

## 표·그림 활용

### 일반 원칙

- 정량 정보는 표로 (비교표, 일정표, 배점 매핑표)
- 프로세스·구조는 다이어그램으로
- 시각 자산 비중을 *높게* — 사업계획서는 **12행당 1표 / 20행당 1그림** 수준이 통상

### 캡션 양식

- 표·그림 캡션은 시각 자산 **바로 다음 줄** 에 배치
- 캡션 형식: **"표 N. 제목"** / **"그림 N. 제목"** (마침표 + 공백 1칸)
- 캡션 번호는 **본문 전체 일련번호** (그림 7 → 19 → 28 ...) — 섹션별 재시작 안 함
- 섹션별 번호(예: "표 2-1") 사용도 가능하나 *한 제안서 내 일관*

### 섹션 도입부 요약표 우선

- 각 섹션 시작 직후 `<표>` 또는 요약표 먼저 → 본문 ❍ 후속
- C 양식의 빈 `<표>` 위치를 그대로 유지하고, 추가 시각자산만 본문 사이에 삽입

### 양식 표 위치 보존

C 양식의 `<표>`·`<그림>` 위치 자리표시자는 *그대로 보존*. 본문은 ❍ 구조로 추가.

## 평가배점 매핑 원칙

- 배점 큰 항목은 본문에서 *명시적으로* 다룸
  - 예: "본 항목은 평가배점 30점에 해당하며, 당사는 다음과 같이 접근합니다."
- 배점 작은 항목은 간결하게 처리
- 누락 항목 없이 모든 평가항목을 본문에서 1회 이상 언급

## KB 인용 패턴

### 검증된 KB 사실은 *문장 단위 그대로 인용*

회사 KB(`dabeeo-profile`)에서 검증된 사실(투자·매출·설립연도·고객사 등)은 **문장 단위로 거의 그대로 인용**. 신뢰성 근거로 활용.

**예시**:
- KB 원문: *"코린도 그룹은 1969년 설립, 인도네시아 재계 20위, 2024년 매출액 약 3조 6천억원"*
- 제안서 인용: *"코린도 그룹은 1969년 설립된 인도네시아 재계 20위의 대기업으로, 2024년 매출 3조6천억원 규모의 기업임"*

### KB와 본문 수치 불일치는 *사전 정정*

KB와 제안서 본문 사이 수치가 다르면 *작성 전*에 KB 갱신 또는 정정. 본문 다 쓰고 검토할 때 발견하면 *재작업* 발생.

### KB에 없는 정보는 *부록 "확인 필요" 항목*

KB에 없는 신규 수치(연구인력·GPU·정부지원 누적 등)는:
1. 가능하면 *KB 보강*해 실제 값 채움
2. 정말 모르면 *부록 D "확인 필요 항목"* 에 모음
3. 본문은 *추상화 표현* (예: "당사 보유 수치 별도 제시")

## RFP 키워드·각주 직접 차용

- RFP 공고의 *정량지표 정의·평가기준 문구*는 제안서 본문에 *그대로 인용*
- 공고 각주를 제안서에서 그대로 복붙 가능 (예: "국민이 체감할 수 있는 박람회·제품설명회 등의 활동 건수")
- 공고 키워드 ("신속상용화", "국민체감활동" 등)는 본문에 *반복 사용*해 정렬도 강조

## 외래어·약어 처리

- **한자/외래어 남용** 회피 — 한글 우선 사용
- **영문 약어는 첫 등장 시 풀이** — "RFP(제안요청서)", "SI(시스템 통합)", "엔드투엔드(End-to-End)", "고객 락인(Lock-in)"
- 단, *분야 표준 약어*는 풀이 없이 직접 사용 가능: AI, GPS, NIR, EVI, IPO, SaaS, MOU, NDA, PoC, ARPU, TRL
- 분야 표준 여부 모호하면 *풀이 우선*

## 회피해야 할 표현

- **과장된 마케팅 표현** — 근거 없는 "세계 최고", "혁신적인", "획기적인" 지양
- **발주처를 깎는 표현** — "귀 기관의 미흡한 부분을 보완" ❌
- **추측·창작 수치** — KB·RFP에 근거 없는 숫자 금지

## 외부 공유본 마스킹

예산·인건비·민감 정보는 *외부 공유본*에서 마스킹:
- 파일명에 *"(인건비 제외)(공유용)"* 등 마스킹 라벨
- 예산표 셀은 *공란*으로 두고 "(인건비 제외)" 표기
- 최종본·공유본 *분리 관리*

## 검토 체크리스트

작성 완료 후 자체 점검:

- [ ] 모든 평가항목을 본문에서 1회 이상 언급
- [ ] 사용한 수치·실적이 KB에서 검증됨
- [ ] 격식체·명사형 종결 일관성 유지
- [ ] 회사 자칭 "당사" 통일
- [ ] 단정형 우선, 추측형(~할 수 있다) 0
- [ ] 모든 정성 주장에 정량 수치 동반
- [ ] ❍ / - / * 3-tier bullet 일관 적용
- [ ] 표·그림에 일련번호 + 캡션 (시각자산 *바로 다음 줄*)
- [ ] 섹션 도입부 요약표 우선
- [ ] 영문 약어 첫 등장 시 풀이
- [ ] 발주처 정식명칭 표기 일관성
- [ ] RFP 공고의 키워드·정의·각주를 본문에 차용
- [ ] KB와 본문 수치 일치 (불일치 시 사전 정정)
- [ ] 외부 공유본 민감 정보 마스킹
