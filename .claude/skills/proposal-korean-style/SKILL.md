# 한국어 제안서 작성 스타일

## 기본 톤

- **격식체 "당사는~합니다"** (반말·평어 금지)
- 1인칭은 "저희"보다 "당사" 선호
- 발주처 호칭: "귀 기관", "발주처", 또는 정식 명칭
- 수동태보다 능동태
  - "~에 의해 수행되어진" ❌
  - "당사가 수행함" ⭕

## 문장 구조

- 한 문장에 하나의 의도 (긴 복문 지양)
- 결론 먼저, 근거 나중 (PREP: Point–Reason–Example–Point)
- 능동·짧고 명확하게

## 표준 구조 (공공 제안서)

```
1. 사업 이해
   1.1 사업 배경
   1.2 사업 목적
   1.3 사업 범위

2. 사업 추진 방안
   2.1 추진 전략
   2.2 추진 방법론
   2.3 단계별 접근

3. 추진 일정·조직
   3.1 추진 일정 (WBS, 간트차트)
   3.2 수행 조직도
   3.3 핵심 인력 이력

4. 사업 수행 역량
   4.1 회사 소개
   4.2 유사 실적
   4.3 보유 기술

5. 품질·보안 관리
   5.1 품질관리 방안
   5.2 정보보안 관리
   5.3 변경관리 절차

6. 부록
   - 인증서 사본
   - 기술 자료
   - 회사 일반 자료
```

## 표·그림 활용

- 정량 정보는 표로 (비교표, 일정표, 배점 매핑표)
- 프로세스·구조는 다이어그램으로
- 모든 표·그림에 일련번호 + 캡션
  - 예: "표 2-1. 추진 일정"
  - 예: "그림 3-2. 수행 조직도"

## 평가배점 매핑 원칙

- 배점 큰 항목은 본문에서 *명시적으로* 다룸
  - 예: "본 항목은 평가배점 30점에 해당하며, 당사는 다음과 같이 접근합니다."
- 배점 작은 항목은 간결하게 처리
- 누락 항목 없이 모든 평가항목을 본문에서 1회 이상 언급

## 회피해야 할 표현

- **한자/외래어 남용** — 한글 우선 사용
- **영문 약어는 첫 등장 시 풀이** — "RFP(제안요청서)", "SI(시스템 통합)"
- **과장된 마케팅 표현** — "세계 최고", "혁신적인", "획기적인" 등 지양
- **발주처를 깎는 표현** — "귀 기관의 미흡한 부분을 보완" ❌

## 검토 체크리스트

작성 완료 후 자체 점검:

- [ ] 모든 평가항목을 본문에서 언급했는가
- [ ] 사용한 수치·실적이 KB에서 검증되었는가
- [ ] 격식체 일관성 유지
- [ ] 표·그림에 번호와 캡션이 모두 있는가
- [ ] 약어가 첫 등장 시 풀이되었는가
- [ ] 발주처 정식명칭 표기 일관성
